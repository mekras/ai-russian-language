# Примеры гибридных форм

Используй этот файл как набор образцов для однотипных замен. Он нужен, чтобы
узнавать форму ошибки, а не ограничиваться перечисленными словами.

## Имя технологии плюс русское родовое слово

- `PHP-решение` → `решение на PHP`
- `TypeScript-компонент` → `компонент TypeScript`
- `Docker-образ` → `образ Docker`
- `JSON-файл` → `файл JSON`
- `APM-скрипт` → `скрипт APM`
- `Inspect-модуль` → `модуль Inspect`
- `MCP-инструменты` → `инструменты MCP`
- `JSON-сценарии` → `сценарии JSON`
- `Python-валидаторы` → `валидаторы Python`
- `Keycloak-реализация` → `реализация через Keycloak`

## Рабочий термин на английском внутри русской фразы

- `auth-contract` → `контракт аутентификации`
- `token claims` → `утверждения токена`
- `dev-зависимость` → `зависимость для разработки`
- `sample-объект` → `объект-пример` или `пример объекта`
- `samples` → `примеры`
- `runner` → `средство запуска`
- `oracle` → `эталон`, `эталонный результат`, `эталонная проверка`
- `judge` → `проверка`, `сравнение`
- `trigger` → `срабатывание`, `условие срабатывания`
- `result` → `результат`
- `eval` → `проверка`, `оценка`

## Английские режимы доступа

- `read-only режим` → `режим только для чтения`
- `bounded read-only` → `ограниченный режим только для чтения`
- `bounded read-only` → `ограниченный запуск без записи`
